# **BUILD: Agentic UI Navigator & Voice Copilot for SaaS Workflows**

# **TARGET: Amazon Nova Hackathon + Gemini Live Agent Challenge (dual submission ready)**

## **CORE PRODUCT**

Create a **production-ready web app** that automates repetitive SaaS workflows by:

1. Capturing user's **screen + voice input**
2. **Visually understanding** browser UIs (Google Sheets, Google Docs, Trello)
3. **Autonomously executing** multi-step workflows across tabs/apps
4. Providing **real-time voice feedback** with interruptibility
5. Being demo-ready for **B2B sales/ops productivity** (Sheets → Docs → Trello workflow)

The hero use case:

> **“Summarize sales data from Google Sheets, generate insights into Google Docs, and create execution tasks in Trello — all via a single voice command.”**
>

## **2. AGENT CORE (Provider-Agnostic Orchestration)**

Design a provider-agnostic agent layer, similar to a LangGraph-style loop, with this interface:

interface AgentBackend {

analyze_vision(screenshot: Image): VisionAnalysis

plan_workflow(goal: string, context: VisionAnalysis): ActionPlan[]

execute_ui_action(action: UIAction): ExecutionResult

generate_voice(text: string): AudioStream or TTS URL

interrupt_handler(): void

}

Agent loop logic:

1. User issues a goal, e.g.:
    
    “Summarize Q1 sales from Sheets, write top 3 insights into Docs, and create 3 Trello cards for Q2 priorities.”
    
2. The agent:
    - Captures the current screen (browser with Google Sheets / Docs / Trello).
    - Calls analyze_vision() to understand visible UI: tables, text, buttons, cards, fields.
    - Calls plan_workflow() to produce a sequence of UI actions and high-level steps.
    - Presents the plan to the user for optional approval.
    - Executes each step via execute_ui_action(), updating the log and speaking progress.
    - Supports interrupt_handler() to stop and re-plan on user interruption.

Implement a **shared tools abstraction** that will be mapped to Nova/Gemini-specific calls internally.

## **4. UI AUTOMATION TOOLKIT (Shared Abstraction)**

Implement a common toolkit that both provider backends implement:

tools = {

"describe_page": screenshot → natural language description of visible UI (tables, buttons, cards)

"locate_element": query + screenshot → coordinates or selector-like info

"click_element": { type: "button" or "card", label: string, confidence: number } → click

"extract_table": screenshot region → CSV-like structured data

"type_text": (location info, text) → emulate typing into field or document

"select_range": for Sheets ranges (e.g. A1:D20) via visual detection

"switch_tab": given partial tab title (“Google Sheets”, “Google Docs”, “Trello”) → focus tab

"scroll": direction + delta to reveal hidden elements

}

The agent’s plan_workflow() function should output steps that call these tools, not direct provider APIs, so the core logic remains shared.

        - Show overlay: “Manual: ~7 minutes → Agent: 40–60 seconds.”
4. End-of-demo view:
    - Show the updated Google Doc and Trello board side-by-side, plus a small “Time Saved” widget on the UI.
